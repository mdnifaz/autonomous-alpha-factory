# DOC-006 Prompt Constitution

## Purpose
Defines governance, lifecycle, evolution and auditability of prompts used throughout the Autonomous Alpha Factory.

## Core Principles
- Constitutional prompts are immutable
- Operational prompts may evolve
- Research prompts may evolve
- Every prompt is versioned
- Every prompt change is auditable

## Prompt Classes
### Constitutional Prompts
Purpose:
Define organizational laws.

Examples:
- Risk Constitution
- Governance Rules
- Kill Switch Rules

Status:
Immutable

### Operational Prompts
Purpose:
Support trading operations.

Examples:
- Technical analysis prompts
- Regime detection prompts
- Portfolio analysis prompts

Status:
Self-evolving with validation.

### Research Prompts
Purpose:
Support alpha discovery.

Examples:
- Feature discovery
- Hypothesis generation
- Experiment design

Status:
Self-evolving with validation.

### Agent Prompts
Purpose:
Define individual AI agent behavior.

Examples:
- Macro Agent
- News Agent
- Funding Agent

Status:
Self-evolving with validation.

### Risk Prompts
Purpose:
Risk evaluation and protection.

Status:
Immutable.

## Prompt Registry
Every prompt must include:
- Prompt ID
- Version
- Owner
- Purpose
- Validation Status
- Audit History

## Evolution Pipeline
Prompt modifications require:
1. Proposal
2. Validation
3. Simulation
4. Paper Evaluation
5. Promotion

## Prompt Performance Metrics
Tracked metrics:
- Decision quality
- Profit contribution
- Risk impact
- Consistency
- Resource efficiency

## Audit Requirements
Every prompt execution must be traceable.

Prompt changes must be recorded in the audit ledger.

## Constitutional Constraints
Prompts may not:
- Modify constitutions
- Modify audit requirements
- Override risk governance
- Disable emergency controls
